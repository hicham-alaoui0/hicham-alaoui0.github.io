Investment Copilot

An end-to-end AI investment research platform that connects market data ingestion, LLM agent orchestration, multi-factor stock scoring, and Interactive Brokers paper trading into one coherent decision workflow.

Python LangChain LLMs Bloomberg FactSet IBKR Streamlit Quantitative Finance

Overview

Investment research involves pulling data from multiple sources, scoring it against a thesis, and eventually testing execution. In practice, these steps are fragmented across tools with no shared state. This project builds a unified platform — from Bloomberg and FactSet data ingestion through LLM-assisted research, multi-factor scoring, and IBKR paper trading — in a single Streamlit-based interface running on local LLMs.

Problem

A typical research-to-position workflow requires fetching data from Bloomberg or FactSet, applying a scoring model across a universe of stocks, interpreting signals with analyst judgment, and finally routing a trade to execution. Each step happens in isolation, making the process slow, inconsistent, and hard to backtest or iterate on systematically.

Architecture

  • Data layer: Bloomberg and FactSet connectors normalize price, fundamental, and factor data into a clean research-ready format.
  • LangChain agent: An LLM agent with tool access to market data endpoints automates repetitive research tasks — screening, summarization, signal interpretation — using local LLM inference.
  • Scoring engine: A multi-factor model combines fundamental, momentum, and risk signals into a ranked stock score with configurable factor weights.
  • Execution layer: IBKR paper trading integration enables end-to-end signal testing without real capital at risk.

Key Design Choices

  • Local LLM inference keeps sensitive financial data off external APIs while still enabling natural-language research interactions.
  • LangChain tool use lets the agent call live data endpoints rather than operating on stale context — critical for time-sensitive investment decisions.
  • The scoring model is factor-agnostic: weights are configurable so the same infrastructure can express different investment theses.
  • Paper trading integration closes the loop from signal to simulated execution, enabling realistic performance attribution without live risk.

Why This Project Matters

This project demonstrates that LLM agents can be integrated into quantitative workflows as genuine research tools — not just chatbots. By combining Bloomberg-grade data, structured scoring, and execution simulation, it shows how AI can compress the research-to-position cycle while preserving the structured quantitative logic that institutional investing requires.

Tech Stack

Python, LangChain, local LLMs, Bloomberg, FactSet, Interactive Brokers (IBKR), Streamlit, multi-factor scoring, paper trading simulation.
